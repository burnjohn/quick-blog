# brainstorm

Start a brainstorming session for a new feature or significant change.

## Instructions

Read and follow the brainstorming skill at `.cursor/skills/brainstorming/SKILL.md` exactly as presented.

The skill will guide you through:
1. Understanding the current project state
2. Asking questions one at a time
3. Proposing 2-3 approaches with trade-offs
4. Presenting design in digestible chunks for validation
5. Writing a design document to `docs/plans/`

## Important

- Do NOT skip straight to code — design first
- Ask ONE question at a time
- Always offer alternatives before deciding
- Keep it simple — YAGNI
