# Commands

Slash commands for common workflows. Invoke with `/command-name` in Cursor chat.

## Catalog

| Command | Description |
|---------|-------------|
| [/commit](commit.md) | Generate commit messages following conventional commits format |
| [/create-branch](create-branch.md) | Create Git branches with naming conventions |
| [/create-pr](create-pr.md) | Create GitHub PRs with file-specific descriptions |
| [/brainstorm](brainstorm.md) | Start a brainstorming session for a new feature |
| [/plan](plan.md) | Create a step-by-step implementation plan |

## Shared Conventions

All Git commands follow conventions from the **git-workflow** skill (`.cursor/skills/git-workflow/SKILL.md`):
- Conventional commit types and message format
- Branch naming patterns
- PR description rules
- Push policy (never push unless explicitly asked)

## Usage

Type `/commit`, `/create-branch`, `/create-pr`, `/brainstorm`, or `/plan` in the Cursor chat.
